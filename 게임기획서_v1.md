## 1. 프로젝트 개요

### 1.1 한 줄 정의
현실적인 고양이 양육 시뮬레이션. 예쁜 고양이 게임이 아니라, 실제로 키울 때 생기는 어려움을 그대로 경험하게 하는 게임.

### 1.2 목적
- 고양이를 키우고 싶은 사람에게 현실적인 준비 경험 제공
- 충동 입양 방지 및 유기 예방
- 실제 입양 후 올바른 케어 방법 자연스럽게 습득

### 1.3 타겟 유저
- 1순위: 고양이 입양을 고민 중인 20~30대
- 2순위: 이미 키우고 있지만 정보가 부족한 초보 집사
- 3순위: 고양이를 좋아하지만 키울 수 없는 환경의 유저 (대리 경험)

### 1.4 핵심 메시지
"힘들다. 그런데 그래서 더 소중하다."
어려움만 강조하지 않고, 잘 돌봤을 때의 보람과 유대감도 함께 전달하여 입양 포기가 아닌 책임감 있는 입양을 유도한다.

### 1.5 플랫폼 전략
- 1단계: 웹 (MVP 출시, 빠른 피드백 수집)
- 2단계: 앱 (iOS / Android, Capacitor로 전환)

## 2. 게임 구조

### 2.1 기본 플레이 구조
현실 시간과 1:1로 연동되는 **완전 리얼타임 시뮬레이션**.
플레이어가 게임을 종료해도 고양이는 실시간으로 배고파지고, 화장실이 더러워지고, 심심해진다.

**리얼타임을 선택한 이유**
이 게임의 목적은 현실적인 부담감을 체험시키는 것이다. 관심이 식어서 게임을 방치하는 것 자체가 하나의 메시지다. "그 정도면 고양이를 키우지 마세요."

| 구분 | 내용 |
|---|---|
| 시간 단위 | 현실 시간 = 게임 시간 (1:1 완전 리얼타임) |
| 스탯 악화 속도 | 현실 고양이와 동일한 속도로 설계 (1일 미급여 → 위험, 3일 → 심각) |
| 하루 접속 목표 | 2~3회 (강제 아님, 자연스럽게 유도) |
| 알림 | 아침 급여 / 저녁 급여 / 저녁 루틴 묶음 / 돌발 이벤트 (주 2~3회) |

**방치 시 시각 연출 — 죄책감 시스템**

| 방치 기간 | 화면 연출 |
|---|---|
| 12시간 | 밥그릇 비어있음, 고양이 불만 표정 이미지 |
| 1일 | 화장실 가득 참, 털 약간 엉킴, 웅크린 자세 |
| 2일 | 화면 채도 낮아짐, 고양이 구석에 숨은 이미지 |
| 3일 이상 | 어두운 화면 + "많이 기다렸어요" 텍스트, 건강 수치 경고 |

**취침 모드 설정**
- 취침 모드 ON: 설정 시간대 알림 차단 → 다음날 아침 고양이 허기·스트레스 누적 상태로 시작
- 취침 모드 OFF: 돌발 알림 가능하지만 고양이 상태 더 잘 유지됨
- 현실에서도 자다가 일어나 케어하거나, 자고 일어나면 밥그릇이 비어있거나 둘 중 하나인 것처럼

### 2.2 게임 루프 (핵심 사이클)

```
[매일 반복]
아침 기상 → 사료 급여 → 화장실 확인 → 낮 상태 체크
→ 저녁 급여 → 놀아주기 → 취침

[주간 이벤트]
빗질 / 발톱 정리 / 용품 점검 / 병원 예약

[월간 이벤트]
예방접종 / 심장사상충 예방약 / 정기 검진

[랜덤 돌발 이벤트]
새벽 울음 / 구토 / 설사 / 가구 파손 / 이상 행동 감지
```

### 2.3 게임 진행 단계
고양이 나이에 따라 케어 내용이 달라진다.

| 단계 | 기간(실제 시간) | 특징 |
|---|---|---|
| 아기 고양이 | 입양 후 2주 | 사회화, 화장실 교육, 예방접종 시작 |
| 성묘 | 2주 ~ 6개월 | 중성화 이벤트, 루틴 확립, 질병 이벤트 시작 |
| 노령묘 | 6개월 이후 | 건강 이벤트 증가, 의료비 부담 증가, 사별 엔딩 준비 |

### 2.4 게임의 두 핵심 축

이 게임의 모든 어려움은 두 가지 축에서 발생한다. 이 두 축이 맞물리는 지점이 핵심 현실감이다.

**축 1 — 시간적 부담 (꾸준한 루틴)**

화장실 청소 · 빗질 · 놀아주기. 한 번 빠진다고 큰일 나는 게 아니라 꾸준히 안 하면 서서히 나빠지는 구조. 이 세 가지에 미니게임이 붙어서 매번 실제로 행동해야 하며, 매일 반복되는 피로감을 느끼게 설계한다.

**축 2 — 경제적 부담 (비용)**

- 예측 가능한 고정 지출: 사료, 모래, 기본 용품 (매달 나가는 돈, 예산 관리 필요)
- 예측 불가능한 돌발 지출: 병원비 (갑자기 터지는 큰 금액, 이 게임의 핵심 충격 포인트)

좋은 사료를 쓸수록 건강에 유리하지만 비용이 올라가는 선택 구조.
준비가 안 된 플레이어는 잔액 부족으로 치료를 못 하거나 저렴한 옵션을 선택해야 하는 상황이 생긴다.

**두 축의 연결**
```
루틴 방치 → 건강 악화 → 병원비 발생 → 경제적 충격
저가 사료 지속 → 건강 서서히 하락 → 결국 더 큰 병원비
```

### 2.5 엔딩 시스템
단순 점수제가 아닌, 양육 방식에 따른 분기 엔딩.

| 엔딩 유형 | 조건 | 메시지 |
|---|---|---|
| 최고 집사 엔딩 | 행복도/건강 장기간 고수준 유지 | 고양이와의 깊은 유대, 노령까지 함께한 이야기 |
| 평범한 집사 엔딩 | 보통 수준 관리 | 서툴지만 함께한 일상 |
| 번아웃 집사 엔딩 | 스트레스 수치 한계 도달 | 현실적 한계 인정, 재입양 고려 메시지 |
| 이별 엔딩 (공통) | 노령묘 사망 | 반려동물 사별의 슬픔, 함께한 시간의 의미 |

## 3. 품종 시스템

### 3.1 선택 가능 품종 (MVP 6종)

#### 코리안 숏헤어 — 밸런스형 / 추천 난이도: ★★
- 특징: 한국 토종, 전반적으로 무난. 튜토리얼 추천 품종
- 선천 질병: 없음

#### 스코티시 폴드 — 의료비 폭탄형 / 추천 난이도: ★★★
- 특징: 귀여운 외모 뒤의 유전적 질환. 관절염이 거의 필연적으로 발생
- 선천 질병: 골연골 이형성증 (발생 확률 80%)

#### 래그돌 — 애교쟁이 장모종 / 추천 난이도: ★★★
- 특징: 관심을 많이 줘야 하고 털 관리 부담이 크다. 심장 검진 필수
- 선천 질병: 비후성 심근증 HCM (발생 확률 40%)

#### 러시안 블루 — 도도한 독립형 / 추천 난이도: ★★★
- 특징: 건강하지만 신뢰를 쌓기까지 오래 걸린다. 환경 변화에 극도로 예민
- 선천 질병: 없음 (단, 스트레스 누적 시 방광염 이벤트 다발)

#### 벵갈 — 에너지 하드모드 / 추천 난이도: ★★★★★
- 특징: 놀이 미충족 시 가구 파손·소음 이벤트 연속 발생. 초보에게 비추천
- 선천 질병: HCM (25%) / 진행성 망막위축 PRA (20%)

#### 페르시안 — 극한 그루밍형 / 추천 난이도: ★★★★
- 특징: 매일 빗질 필수. 호흡기·눈물자국 관리가 일상. 신장 질환 주의
- 선천 질병: 다낭성 신장병 PKD (30%) / 호흡기 문제 (70%)

## 4. 스탯 시스템

### 4.1 변동 스탯 (플레이어 행동으로 오르내리는 값)

| 스탯 | 범위 | 위험 구간 | 주요 영향 요소 |
|---|---|---|---|
| 현재 건강 | 0~100 | 30 이하 | 사료 질, 병원 방문, 그루밍, 청결 |
| 비만도 | 0~100 | 70 이상 | 사료량, 간식, 활동량 |
| 스트레스 | 0~100 | 70 이상 | 놀이 부족, 화장실 불결, 환경 변화, 애정 부족 |
| 위생 | 0~100 | 30 이하 | 화장실 청소 주기, 빗질 여부 |
| 행복도 | 0~100 | 30 이하 | 건강·스트레스·위생·신뢰도의 종합 결과값 |
| 신뢰도 | 0~100 | — | 꾸준한 케어, 스킨십, 놀이로 누적 상승 |
| 허기 | 0~100 | 80 이상 | 급여 주기 및 양 |

### 4.2 스탯 상호작용 공식

**허기 증가 속도**
```
시간당 허기 증가 = 5 + (활동성 × 0.3)
```
활동적인 품종일수록 더 빨리 배고파진다.

**비만도 변화**
```
비만도 변화 = (오늘 섭취 칼로리 - 오늘 소모 칼로리) × 비만 경향 계수
```
같은 양을 먹어도 래그돌이 코리안 숏헤어보다 더 빨리 찐다.

**스트레스 누적 속도**
```
스트레스 누적 = 스트레스 민감도 × (화장실 불결도 × 0.3 + 허기 × 0.2 + 놀이 미충족 × 0.3 + 환경 변화 여부 × 0.2)
```

**질병 이벤트 트리거 확률**
```
질병 발생 확률 = (10 - 타고난 건강) × 2% + 스트레스 × 0.1% + 선천 질병 기본 확률
```

**행복도 (종합)**
```
행복도 = (현재 건강 × 0.3) + (100 - 스트레스 × 0.3) + (위생 × 0.2) + (신뢰도 × 0.2)
```

### 4.3 스탯 방치 시 결과

| 방치 스탯 | 경고 단계 | 위험 단계 결과 |
|---|---|---|
| 허기 80 이상 | 울음 이벤트 | 건강 빠른 감소 |
| 스트레스 70 이상 | 이상 행동 이벤트 | 화장실 밖 배변, 가구 스크래칭 |
| 위생 30 이하 | 불쾌감 표현 | 피부염·헤어볼 이벤트 |
| 비만도 70 이상 | 느려진 움직임 | 당뇨·관절 이벤트 트리거 |
| 행복도 30 이하 | 우울 상태 진입 | 은둔, 식욕 저하, 신뢰도 하락 |

## 5. 이벤트 시스템

### 5.1 일상 이벤트 (매일 반복)

| 이벤트 | 빈도 | 플레이어 선택지 | 결과 차이 |
|---|---|---|---|
| 아침 사료 급여 | 매일 | 사료 종류·양 선택 | 건강·비만도·편식 반응 |
| 화장실 청소 | 1~2회/일 | 즉시 청소 / 나중에 | 위생·스트레스 영향 |
| 물 교체 | 매일 | 교체 / 패스 | 건강 소폭 영향 |
| 놀이 시간 | 1~2회/일 | 장난감 종류·시간 선택 | 스트레스·신뢰도 영향 |
| 스킨십 | 매일 | 안기·쓰다듬기 | 경계심 따라 반응 다름 |

### 5.2 주간 이벤트

| 이벤트 | 주기 | 미이행 시 결과 |
|---|---|---|
| 빗질 | 3~7회/주 (품종별 상이) | 헤어볼·피부염 이벤트 |
| 발톱 정리 | 2주 1회 | 가구 스크래칭 이벤트 증가 |
| 귀 청소 | 2주 1회 | 외이염 이벤트 |
| 장난감·용품 점검 | 주 1회 | 파손 용품 방치 시 사고 이벤트 |

### 5.3 월간 이벤트

| 이벤트 | 주기 | 비용 | 미이행 시 결과 |
|---|---|---|---|
| 심장사상충 예방 | 월 1회 | 소액 | 감염 이벤트 확률 증가 |
| 벼룩·진드기 예방 | 월 1회 | 소액 | 피부 이벤트 |
| 정기 건강검진 | 6개월 1회 | 중간 비용 | 질병 조기 발견 기회 놓침 |
| 예방접종 | 연 1회 | 중간 비용 | 전염병 이벤트 확률 증가 |

### 5.4 돌발 이벤트 (랜덤)

**경미한 이벤트**
- 새벽 울음 (알림 발송, 무시 시 스트레스 누적)
- 헤어볼 구토 (청소 필요)
- 물건 떨어뜨림 (스트레스 원인 제거 선택지)
- 간식 구걸 (주면 비만, 안 주면 서운함)

**중간 이벤트**
- 설사·구토 반복 (병원 판단 필요)
- 화장실 밖 배변 (스트레스·질병 신호)
- 가구·벽지 스크래칭 (스트레스 또는 스크래처 부족)
- 식욕 감소 (질병·스트레스 신호)
- 발정기 소음 (중성화 전 품종)
- 이상한 자세로 앉기 (관절 이상 신호 — 스코티시 폴드 다발)

**중증 이벤트 (병원 필수)**
- 호흡 이상 (페르시안, 래그돌, 벵갈)
- 혈뇨·배변 장애 (러시안 블루 스트레스성 방광염)
- 구내염 증상
- 체중 급감
- 눈·귀 분비물 과다

**특수 이벤트**
- 이사 이벤트: 스트레스 민감도 높은 품종 집중 타격, 적응 기간 필요
- 여행 부재 이벤트: 케어 서비스 고용 or 지인 부탁 선택지, 비용 발생
- 다묘 추가 이벤트: 합사 과정 이벤트 (영역 싸움, 적응 기간)
- 사별 이벤트: 노령묘 단계 이후 건강 악화 → 최후 선택지 (자연사 or 안락사 고민)

### 5.5 핵심 미니게임 설계

미니게임의 목적은 재미가 아니라 **반복의 피로감과 행동의 무게**를 전달하는 것. 난이도보다 단조로운 반복감을 의도적으로 설계한다.

**① 화장실 청소**
- 방식: 모래 화면을 탭/드래그해서 덩어리를 스쿱으로 걷어내기
- 방치 기간 비례: 덩어리 수가 많아지고 화면이 더 지저분해짐
- 특이 이벤트: 청소 중 고양이가 들어와 방해 (처음부터 다시)
- 연출: 삽질 소리, 모래 소리 사운드 이펙트로 현실감 강화

**② 빗질**
- 방식: 화면을 일정 방향으로 드래그하면 털이 걷힘
- 장모종(래그돌·페르시안): 영역 넓고, 엉킨 구간은 짧게 반복 드래그해야 풀림
- 고양이 스트레스 높을 때: 중간에 피하거나 할퀴기 이벤트 → 처음부터
- 끝나도 며칠 후 또 해야 함을 알림으로 상기 (반복의 무한함 전달)

**③ 놀아주기**
- 방식: 장난감을 화면에서 움직이면 고양이가 따라옴
- 만족 임계치: 활동성 스탯에 비례 (벵갈은 매우 높아서 플레이어가 지침)
- 고양이가 흥미 잃으면: 다시 유도해야 게이지 상승
- 목적: 활동성 높은 품종이 얼마나 많은 에너지를 요구하는지 체감

**④ 병원 데려가기**
- 1단계 — 캐리어 넣기: 고양이가 도망다니고 탭으로 잡아야 함. 경계심 높은 품종일수록 더 많이 도망.
- 2단계 — 진료비 고지: 숫자가 올라가며 최종 금액이 표시되는 연출. 숫자가 올라가는 속도 자체가 충격.
- 효과: 병원을 피하고 싶은 심리를 자연스럽게 만들지만, 안 가면 더 큰일이 난다는 것도 학습.

**⑤ 약 먹이기**
- 방식: 알약을 사료에 숨기거나 직접 입에 넣기 선택
- 직접 먹이기: 고양이가 뱉어내고 도망 → 반복 시도
- 사료에 숨기기: 고양이가 약만 골라낼 수 있음 (랜덤 확률)
- 성공까지 시도 횟수가 플레이어 스트레스를 누적시키는 구조

### 5.6 고양이의 아픔 숨기기 메커니즘
고양이는 아픔을 숨기는 습성이 있다는 현실을 게임 시스템으로 구현.

- 질병 이벤트 발생 후 **일정 기간 동안 증상이 UI에 표시되지 않음**
- 플레이어가 이상 징후(식욕 감소, 움직임 감소 등)를 스스로 발견해야 함
- 정기 검진을 받으면 조기 발견 가능 → 치료비 절감
- 방치 시 중증으로 발전 → 치료비 대폭 증가 또는 사망

## 5-A. 감동 이벤트 시스템 (긍정적 보상)

### 설계 원칙
어려움만 강조하면 "키우지 말라는 게임"이 된다. 실제로 사람들이 힘들어도 버티게 만드는 순간들을 함께 전달해야 한다. 핵심은 **계획할 수 없는 우연성**이다. 플레이어가 요청하거나 예측할 수 없고, 꾸준히 잘 돌봤을 때 발생 확률이 올라가는 랜덤 구조.

감동 이벤트는 **대비 효과**로 작동한다. 힘든 미니게임 직후나 병원비 고지 직후에 터질 때 감정 전달력이 가장 크다. 이 타이밍을 시스템적으로 가중치에 반영한다.

### 감동 이벤트 목록 (신뢰도 단계별 잠금 해제)

**초반 — 신뢰도 30 이하**
- 처음으로 이름을 부르면 돌아봄
- 밥 먹고 나서 잠깐 발치에 앉음
- 창가에서 햇빛 받으며 그루밍하는 모습

**중반 — 신뢰도 30~70**
- 처음으로 무릎에 올라옴
- 꾹꾹이 시작 (안정의 신호)
- 골골 소리를 내며 옆에서 잠듦
- 장난감을 물고 와서 보여줌

**후반 — 신뢰도 70 이상**
- 배를 뒤집어서 보여줌 (완전한 신뢰의 표시)
- 힘든 날 조용히 곁에 와서 앉아있음
- 자다가 발치에 붙어서 같이 잠
- 오랜만에 집에 오면 문 앞에서 기다리고 있음

### 시각·청각 연출

감동 이벤트는 일반 이벤트와 연출 방식을 완전히 분리한다.

```
일반 이벤트: 카드 팝업 (상황 설명 + 선택지)

감동 이벤트: 화면 전환
            → 따뜻한 색감으로 전체 화면 전환
            → 풀스크린 감성 일러스트
            → 짧은 텍스트 한 줄
            → 골골 소리 or 잔잔한 BGM
            → 천천히 페이드아웃
```

텍스트 예시:
> "오늘 유독 힘들었던 날, 냥이가 조용히 옆에 와서 앉았다."
> "처음으로 배를 보여줬다. 믿는다는 뜻이다."
> "무릎이 따뜻해졌다."

### 메인 화면 — 일상 상태 이미지 (상시 노출)

감동 이벤트와 별개로 메인 화면에서 고양이 상태에 따른 이미지가 자연스럽게 표시된다. 귀여운 일상 이미지는 배경 역할이며, 감동 이벤트의 특별함을 해치지 않도록 연출을 분리한다.

| 상태 | 메인 이미지 |
|---|---|
| 행복 | 동그랗게 말려서 낮잠 / 장난감과 놀기 |
| 배고픔 | 밥그릇 앞에 앉아서 쳐다보기 |
| 스트레스 | 귀 뒤로 접힘, 꼬리 흔들기 |
| 아픔 | 웅크리고 눈 반만 뜸 |
| 불만 (화장실) | 화장실 앞에서 울기 |
| 신뢰 높음 | 눈 마주치고 천천히 눈 깜빡이기 (캣 키스) |
| 우울 | 구석에 혼자 앉아 창밖 보기 |

## 7. 기술 스택

### 7.1 확정 스택

| 영역 | 기술 | 이유 |
|---|---|---|
| 프론트엔드 | React + TypeScript | 컴포넌트 기반 UI, 타입 안정성 |

| 상태 관리 | Zustand | 가볍고 직관적 |
| 로컬 저장 | IndexedDB | 실시간 게임 데이터 영속성 |
| 알림 | Web Push API → 앱 전환 후 Native Push | 핵심 현실감 기능 |
| 앱 전환 | Capacitor | 웹 코드 그대로 iOS/Android 빌드 |

### 7.2 개발 단계

**Phase 1 — 웹 MVP (1~3개월)**
- 코어 시스템 구현 (스탯, 급여, 화장실, 놀이)
- 품종 2종으로 시작 (코리안 숏헤어 + 스코티시 폴드)
- 주요 이벤트 20개
- 기본 UI/UX

**Phase 2 — 웹 정식 (3~5개월)**
- 전 품종 6종 추가
- 전체 이벤트 시스템
- 수익 시스템 (인앱 구매 구조 구현)
- 사용자 피드백 반영

**Phase 3 — 앱 출시 (5~8개월)**
- Capacitor로 iOS/Android 빌드
- 네이티브 푸시 알림 구현
- 앱스토어 심사 및 출시

## 9. 비주얼 & 감성 디자인 원칙

### 9.1 일러스트 스타일
**감성 일러스트** (따뜻한 수채화풍) 확정.
- 이유: 이 게임의 핵심이 감정 이입이며, 픽셀·벡터 스타일보다 감성 전달력이 월등히 높다
- AI 이미지 생성(Midjourney 등)으로 품종별 상태 이미지 제작, 비용 효율적
- 품종별로 각 상태 이미지를 별도 제작 (7가지 상태 × 6품종 = 42장 기본 세트)

### 9.2 화면 구성 원칙
- 메인 화면: 중앙 고양이 이미지 + 배경 집 환경 + 하단 스탯 바
- 이벤트: 카드 팝업 (텍스트 + 선택지) — 일반 이벤트
- 감동 이벤트: 풀스크린 일러스트 전환 — 별도 연출
- 미니게임: 해당 액션에 맞는 인터랙티브 화면으로 전환

### 9.3 색상 언어
| 상황 | 색감 |
|---|---|
| 평상시 | 따뜻한 중간 톤 |
| 고양이 행복·건강 | 밝고 따뜻한 색 |
| 스트레스·위험 신호 | 채도 낮아짐, 차가운 톤 |
| 방치 상태 | 어두운 화면, 회색 계열 |
| 감동 이벤트 | 따뜻하고 부드러운 빛 효과 |
